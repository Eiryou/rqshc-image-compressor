# RQSHC V64I

[English](README.en.md)

RQSHC V64Iは、Windows上で動作する画像圧縮の研究・評価用ソフトウェアです。画像を独自形式`.rqi`へ変換し、圧縮後のサイズ、復元品質、処理時間を確認できます。

処理は利用者のPC内で行います。画像を圧縮するために外部のクラウドサービスへ送信する設計ではありません。


## 2026/0719
今回、Webで公開して気軽に圧縮テスト出来る様にWebAssemblyにしてデプロイしました。
その際ローカルで実行するよりWebAssembly化によって圧縮速度やが落ちる、圧縮率低下、品質保持のため現在公開してる.exe (Binary)のソースコードを新しいアルゴリズムやロジックを追加してWebAssemblyでもなんとかWeb公開しても大丈夫な開発が出来ました。

## 現在対応している形式

### 入力

- PNG（`.png`）
- PPM（`.ppm`）
- Windows Bitmap（`.bmp`）

現在、JPEG、WebP、GIF、TIFFなどは入力形式として対応していません。対応していない画像は、事前にPNG、PPM、BMPのいずれかへ変換してください。

### 出力

- RQSHC独自形式（`.rqi`）
- RQI ViewerからPNGへ書き出し可能

RQIは一般的な画像ソフトで直接開けないため、同梱の`RQI_Viewer.exe`を使用します。

## 同梱アプリ

### RQSHC_Observatory.exe

PNG、PPM、BMP画像を読み込み、RQIファイルを作成します。圧縮前後のサイズ、削減率、品質、処理時間を確認できます。

### RQI_Viewer.exe

RQIファイルを開くためのWindows用ビューワーです。

- ファイルをドラッグ＆ドロップして開く
- `Ctrl + マウスホイール`で拡大・縮小
- 通常のホイールで縦スクロール
- `Shift + マウスホイール`で横スクロール
- 左ドラッグで表示位置を移動
- ウィンドウに合わせて表示、100%表示
- PNGとして保存
- クリップボードへコピー
- 日本語／Englishの切り替え

初回はWindowsの表示言語を参考にし、その後は選択した表示言語だけを保存します。画像や閲覧履歴は保存しません。

## 使い方

1. ZIPを展開します。
2. 画像を圧縮するときは`RQSHC_Observatory.exe`を起動します。
3. PNG、PPM、BMPのいずれかを追加して圧縮します。
4. 作成した`.rqi`を確認するときは`RQI_Viewer.exe`へドラッグ＆ドロップします。
5. 必要な場合だけ、ViewerからPNGとして保存します。

重要な画像は、元ファイルを必ず別に保管してください。

## なぜ作ったのか

既存の画像圧縮ライブラリを呼び出すだけではなく、画像データと独自コンテナを自分で扱い、圧縮率、復元品質、処理速度の関係を実際に検証できる環境を作りたいと考えたことが、このプロジェクトの出発点です。

RQSHCは、既存の画像圧縮ソフトを呼び出すだけのフロントエンドではありません。中核となるRQIの圧縮・復号処理は、独自の設計に基づくネイティブC++17で実装し、性能上重要な一部にはx64アセンブリとAVX2最適化を使用しています。

Windows APIやC++標準ライブラリは利用しますが、中核の圧縮・復号を第三者の画像コーデックへ委ねる構成ではありません。画像データとRQIコンテナをバイナリレベルで扱っています。

AIは実装、検証、修正、文書化の支援に利用しています。一方で、設計方針、技術的な制約、評価基準、採用・棄却の判断、公開範囲はプロジェクト設計者が管理しています。

## 現在の実測例

開発時の14画像テストでは、V64Iで次の平均値を記録しました。

- 処理時間：約64.8 ms
- サイズ削減率：約33.49%
- SSIM：約0.999537

画像の内容、解像度、入力形式、CPU、実行環境によって結果は変わります。これはすべての画像で保証される性能ではありません。

## フィードバックについて

特に、次のような情報を歓迎します。

- 圧縮が得意だった画像、不得意だった画像
- 異なるAVX2対応CPUでの処理時間
- RQI Viewerの操作性
- 復元画像の見え方や互換性
- 再現可能な不具合報告
- 測定条件を記載したベンチマーク

良い結果だけでなく、圧縮率が低い例、サイズが増えた例、表示できなかった例も研究上重要です。

## Windows SmartScreenについて

本ソフトウェアは現在コード署名されていないため、初回起動時にWindows Defender SmartScreenが「認識されないアプリ」または「発行元不明」と表示する場合があります。

これは必ずしもマルウェアが検出されたことを意味するものではなく、新しい無署名アプリで発行元情報や利用実績が十分に蓄積されていない場合にも表示されます。必ず公式GitHub Releasesから取得し、公開されているSHA-256と一致することを確認してください。

Windows Defenderや他のセキュリティソフトを無効にする必要はありません。

## 動作環境

- Windows 10 / 11 64-bit
- x64 CPU
- AVX2対応CPUを推奨
- Observatoryでは一般的なデスクトップWebブラウザを使用

## 現在の制限

- 入力はPNG、PPM、BMPのみ
- Windows 64-bit専用
- RQIは独自形式のため、閲覧にはRQI Viewerが必要
- 研究・評価段階のため、すべての画像でサイズが小さくなるわけではない
- 圧縮コアは現時点でクローズドソース
